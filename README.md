## 🧠 My Learning Philosophy
At 42, we don't have teachers or textbooks—only project briefs and peers to battle alongside. Every line of code here strictly follows the school's rigorous formatting standard (Norminette). My rules are simple:
* **Reinventing the wheel is true learning:** Starting from the C standard library, I build core low-level mechanics with my own hands.
* **Know the "how" and the "why":** It's not enough to just make the code run; I need to know exactly what's happening in memory.
* **Embrace the errors:** Every `Segmentation fault` is just a stepping stone to a deeper understanding.

## 🌟 The Magic Showcase (Featured Projects)
Here, code is more than just text. It's the brush that draws graphics and the bridge to converse with the operating system. Here are some of the milestones I've unlocked:

### 🛠️ 1. The Building Blocks (The Foundation of C)
* **[Libft](./libft)**: Rewrote over 40 C standard library functions. **The Magic:** This is where I truly mastered pointer arithmetic and linked lists. It serves as the cornerstone of every project that follows.
* **[ft_printf](./ft_printf)** / **[get_next_line](./get_next_line)**: Unlocking the mysteries of variadic arguments, memory leaks, and file descriptors.

### ⚡ 2. Algorithms & Logic (Making the Machine Smart)
* **[Push_swap](./Push_swap)**: Sorting a random stack of numbers using only two stacks and a highly limited set of instructions. **The Magic:** Pushing algorithm optimization to the absolute limit. Watching the instruction count drop is incredibly satisfying!
